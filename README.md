# zszl_script

`zszl_script` 是一个面向 **Minecraft Forge 1.12.2** 的客户端自动化 Mod，也是一个围绕“**路径动作 + 触发器 + 序列动作 + 变量系统 + 数据包能力 + 自定义 GUI**”构建起来的完整脚本系统。  
它并不是只做几个固定功能的简单辅助，而是把动作编排、事件触发、数据捕获、条件判断、配置持久化、导入导出、分享码和可视化编辑整合到同一套框架里，目标是覆盖用户 **99% 的自动化与脚本化需求**。

当前仓库主线代码面向：

- Minecraft `1.12.2`
- Forge `14.23.5.2860+`
- Java `8`

## 介绍

这套脚本系统的核心思路，是把复杂需求拆成可以自由组合的脚本积木：

- **自定义路径动作**：把移动、交互、等待、点击、条件判断、功能开关、消息发送、目标定位等行为做成可配置动作。
- **序列动作编排**：多个动作可以组成步骤，再由多个步骤组成完整路径序列，支持复用、嵌套调用、分类管理和持久化保存。
- **触发器驱动执行**：脚本不只是“手动点一下开始”，还可以由聊天、数据包、GUI 打开、低血量、计时器、死亡、重生、区域变化、背包变化、实体接近等事件自动触发。
- **变量系统**：支持全局变量、序列变量、局部变量、临时变量，可在动作之间传递上下文、记录状态、做条件分支。
- **数据捕获与发送**：支持数据包采集、过滤、字段提取、拦截规则、快照查看、序列联动，以及主动发送数据包，满足更深层的自动化需求。
- **完整可视化 GUI**：内置路径管理器、动作编辑器、节点编辑器、变量管理、按键管理、配置页、抓包界面、配置分享与导入界面，不依赖外部文本配置也能完成大部分脚本搭建。

换句话说，`zszl_script` 不是“写死的一套功能”，而是一套可以让用户按自己需求搭建逻辑的 **通用脚本平台**。

## 核心能力

### 1. 路径与动作系统

项目提供完整的路径序列系统，支持：

- 内置路径与自定义路径并存
- 路径分类、子分类、隐藏控制
- 动作级参数编辑、动作说明、动作验证
- 路径执行进度快照、执行日志、失败处理
- 动作模板、动作库、最近使用与收藏

动作层不只覆盖普通移动，还包括界面交互、目标捕获、等待条件、功能切换、背包操作、目标定位、调用其他序列等复杂流程，因此能拼装出远超“单功能挂机”的脚本逻辑。

### 2. 触发器与事件驱动

系统内置两套可扩展的触发思路：

- 传统序列触发
- 节点图触发

目前代码中已经覆盖的触发来源包括：

- 聊天消息
- 数据包
- GUI 打开
- 低血量
- 定时器
- 死亡 / 重生
- 区域变化
- 背包变化 / 背包已满
- 附近实体

这意味着脚本可以从“按键启动”升级成“看到事件就自动响应”的自动化系统。

### 3. 变量与数据流系统

变量系统是这套脚本真正强大的地方之一。项目支持：

- `global` 全局变量
- `sequence` 序列变量
- `local` 局部变量
- `temp` 临时变量

变量既可以手动赋值，也可以由动作动态捕获，例如：

- GUI 标题
- 附近实体
- 方块信息
- 背包 / 快捷栏内容
- 实体列表
- 记分板
- GUI 元素
- 屏幕区域
- 数据包字段

有了这套数据流能力，脚本就不再只是“机械执行固定步骤”，而是可以根据运行时环境做判断、等待、分支和联动。

### 4. 数据包系统

项目内置了比较完整的数据包工具链，包含：

- 收包 / 发包采集
- 黑白名单过滤
- 抓包列表查看
- 数据包快照历史
- 字段提取规则
- 拦截规则
- 与路径动作联动的等待条件
- 主动发送数据包

这部分能力让脚本不只停留在表层界面操作，还能围绕协议数据构建更细粒度、更稳定的自动化逻辑。

### 5. 节点编辑与图形化脚本

除了传统序列编辑器，项目还实现了节点图脚本系统，支持：

- 节点图存储与运行
- 触发节点绑定
- 节点参数编辑
- 图导入 / 导出
- 图差异对比
- 草稿与审计日志

这让复杂逻辑除了线性序列，还能以图形化方式进行组织，更适合事件驱动、条件分流、多分支联动的脚本场景。

### 6. 配置、分享与持久化

为了让脚本真正可长期使用，项目还补齐了完整的工程化能力：

- Profile 配置档管理
- 配置文件读写与持久化
- 分享码导出 / 导入
- 导入预览与合并策略
- 按键绑定管理
- 服务器功能可见性控制
- 分类与界面布局持久化

这意味着用户做出来的脚本和配置不仅能保存，还能迁移、分享、复用和按不同场景切换。

### 7. 其他自动化与扩展能力

除脚本框架本体外，项目还集成了大量业务模块与通用能力，例如：

- 自动进食、自动拾取、自动跟随、自动逃脱、自动技能、自动钓鱼、自动装备等处理器
- 仓库、精炼、邮件、快捷兑换、物品过滤等业务界面和逻辑
- 移动、渲染、世界、杂项等“其他功能”分组管理
- 内嵌导航与 ShadowBaritone 相关能力
- 必要时结合 AutoHotKey 的外部输入模拟能力

这些模块既可以单独使用，也可以被路径动作和触发器编排进更大的自动化流程里。

## 为什么说它能覆盖用户 99% 的需求

因为这不是“给几个固定按钮”的脚本，而是一套可以自由组合的系统：

- 你可以用 **路径动作** 描述要做什么
- 用 **触发器** 决定什么时候做
- 用 **序列 / 节点图** 组织流程
- 用 **变量系统** 存数据、做判断
- 用 **数据包捕获与发送** 解决更深层的交互问题
- 用 **自定义 GUI、配置档、分享码、导入导出** 让脚本真正可维护、可复用

因此，无论是日常挂机、复杂交互、条件化执行、协议级联动、配置分发，还是大体量的个人脚本工程，这个项目都不是“只能做一部分”，而是尽量提供一整套完整的脚本基础设施。
