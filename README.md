它不仅仅是辅助工具，而是一个可扩展的**通用脚本自动化平台**。

- 🧱 Minecraft：`1.12.2`
- 🔧 Forge：`14.23.5.2860+`
- ☕ Java：`8`

## 🧠 核心设计理念

将复杂自动化需求拆解为可组合的“脚本积木”：

- 🎯 **动作（Action）**：定义“做什么”
- ⚡ **触发器（Trigger）**：定义“什么时候做”
- 🧩 **序列 / 节点图（Flow）**：定义“如何组织流程”
- 🧷 **变量系统（Variables）**：定义“数据如何流动”
- 📡 **数据包系统（Packet）**：定义“底层交互能力”
- 🖥️ **GUI 编辑器**：定义“如何可视化构建”

👉 最终目标：实现**高度自由的自动化脚本构建系统**

---

## 🧭 🧩 路径与动作系统

支持完整的动作编排系统：

### ✨ 特性

- 📁 自定义路径 + 内置路径
- 🗂️ 分类 / 子分类管理
- 🔁 动作可嵌套调用
- 🧪 动作参数配置 + 校验
- 🧾 执行日志 & 失败处理
- ⭐ 最近使用 / 收藏系统
- 📦 动作模板库

### 🧱 支持动作类型

- 🚶 移动 / 跟随
- 🖱️ 点击 / 交互
- ⏳ 等待 / 条件等待
- 💬 消息发送
- 🎯 目标定位
- 🎒 背包操作
- 🔀 流程调用

## ⚡ 🔔 触发器系统（事件驱动）

脚本不再依赖“手动启动”，而是自动响应世界事件：

### 📡 触发来源

- 💬 聊天消息
- 📦 数据包事件
- 🖥️ GUI 打开
- ❤️ 低血量
- ⏱️ 定时器
- ☠️ 死亡 / 重生
- 📍 区域变化
- 🎒 背包变化 / 满背包
- 👾 附近实体检测

👉 支持：
- 传统序列触发
- 节点图触发

## 🔣 📊 变量与数据流系统

支持多级变量体系：

- 🌍 `global` 全局变量
- 🔁 `sequence` 序列变量
- 📍 `local` 局部变量
- ⚡ `temp` 临时变量

### 📥 可捕获数据

- GUI 标题 / 元素
- 方块信息
- 实体列表
- 背包 / 快捷栏
- 记分板数据
- 屏幕区域信息
- 数据包字段

👉 支持：
- 条件判断
- 数据传递
- 状态保存
- 动态分支逻辑
